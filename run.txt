To run this project, you will need **Node.js** (v18+ recommended) and a **MySQL Server** installed. The project has a Node.js Express backend and a React/Vite frontend.

---

### Step 1: Database Setup
The SQL dump file `erpdata.sql` in the root folder contains data insertion seeds but **does not** contain table creation schema. Therefore, you must sync the database models first before importing the data.

1. Start your **MySQL Server**.
2. Create an empty database for the project (e.g., named `record2`):
   ```sql
   CREATE DATABASE record2;
   ```

---

### Step 2: Backend Configuration & Start
1. Navigate to the `backend` directory in your terminal.
2. Install the backend dependencies:
   ```bash
   npm install
   ```
3. Since different files in the backend search for `.env` or `config.env`, create **both** files in the `backend/` directory with the identical contents below:
   
   Create `backend/.env` and `backend/config.env` containing:
   ```env
   PORT=4000
   DB_HOST=127.0.0.1
   DB_USER=your_mysql_username
   DB_PASSWORD=your_mysql_password
   DB_NAME=record2
   JWT_SECRET=some_random_secret_key_here
   NODE_ENV=development
   # If you have Redis running (optional, system falls back to in-memory if offline):
   # REDIS_HOST=127.0.0.1
   # REDIS_PORT=6379
   ```

4. **Sync the Database Schema** to generate the MySQL tables from Sequelize models:
   ```bash
   node Scripts/sync.js
   ```

5. **Import the Seed Data** using the `erpdata.sql` file in the root directory (ensure you run this from your MySQL terminal/tool or import it manually):
   ```bash
   mysql -u your_mysql_username -p record2 < ../erpdata.sql
   ```

6. **Start the Backend Server** (in development mode with automatic restarts):
   ```bash
   npm run server
   ```
   *The backend will boot up at `http://localhost:4000`.*

---

### Step 3: Frontend Configuration & Start
1. Navigate to the `frontend` directory in a new terminal window.
2. Install the frontend dependencies:
   ```bash
   npm install
   ```
3. *(Optional)* Create a `.env` file in the `frontend` directory if you wish to adjust paths, though the defaults are set to point to the correct URL:
   ```env
   VITE_BACKEND_URL=http://localhost:4000
   VITE_DEBUG_API=true
   ```
4. **Start the Frontend Client**:
   ```bash
   npm run dev
   ```
   *The React development server will start (usually on `http://localhost:5173`).*

---

### Credentials from SQL Seed
Once running, you can log in using one of the users seeded in `erpdata.sql`. For instance:
* **Admin Login**:
  * **Email**: `admin_user@nec.edu.in`
  * **Password**: *Refer to password hash or reset it if needed. Check `backend/Scripts/hashpassword.js` if you want to generate a hashed password to insert directly into the DB.*
